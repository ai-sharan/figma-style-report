Influence Hub — Project Report 2025
Introduction

Influencer marketing is no longer a luxury reserved for large corporations. Local cafés, boutique clothing stores, fitness studios, and small service businesses now compete for attention in the same digital spaces — yet they have no structured, affordable channel to connect with the content creators who can amplify their voice.

Influence Hub was built to close that gap. It is a full-stack web application where local businesses can discover influencers, propose collaborations with defined deliverables and budgets, and complete the entire engagement — from escrow payment to content delivery — without leaving the platform.

This report documents the design decisions, technical architecture, feature set, and the engineering journey behind the platform, built solo as a final-year BCA capstone project.

Built for a market that exists but lacks the right tools

Small and medium businesses struggle to find influencers they can trust, negotiate with, and pay safely. Meanwhile, micro-influencers with loyal local audiences have no centralised platform to find brand deals, track deliverables, or receive payments securely.

Influence Hub solves both sides of this problem with a dual-role platform. A Business account can search for influencers, view public profiles, send collaboration requests with a budget and campaign brief, and hold payment in simulated escrow until deliverables are accepted.

An Influencer account can build a portfolio, set niche and city, receive and accept collaboration offers, submit content for review, and track payments — all within the platform.

An Admin account monitors platform health, manages disputes, and resolves contested collaborations with access to full messaging history.

A modern, industry-standard MERN stack with automation layer

The technology choices were guided by real-world industry adoption, free-tier availability for student deployment, and fit-for-purpose design — not fashionable trends.

Frontend: React.js with Vite for fast HMR and build performance. Tailwind CSS for utility-first styling with a purple/lavender design system. Component pages cover every flow from onboarding to admin dispute resolution.

Backend: Node.js + Express.js REST API with JWT authentication, Helmet security headers, and express-rate-limit to prevent abuse. Multer handles file ingestion before Cloudinary upload.

Database: MongoDB with Mongoose ODM. Collections cover Users, BusinessProfiles, InfluencerProfiles, Collaborations, Messages, and Notifications — all with relational references.

Automation: n8n self-hosted workflow engine handles time-triggered events like auto-releasing escrow payments when a business approval deadline passes.

All services run locally in development. MongoDB Atlas used for cloud deployment.

Authentication, discovery, and full collaboration lifecycle

Email Authentication: Full JWT-based auth with Nodemailer-powered email verification. Users must verify their email before accessing the platform. Password reset via tokenised email link. Bcrypt password hashing throughout.

Profile Onboarding: Businesses fill in industry type, city, and upload photos. Influencers set niche, city, follower count, and build a portfolio with Cloudinary-hosted media — all handled via Multer + Cloudinary pipeline.

Collaboration Lifecycle: The platform tracks five states — Pending → Active → Delivered → Completed / Disputed. Businesses send requests with budget and brief. Influencers accept or decline. Delivery triggers business review. The admin resolves disputes with full context.

Simulated Escrow: Payment is held in a MongoDB-tracked escrow field at request time. It is released to the influencer on approval, refunded on rejection, or auto-released by n8n after a deadline. No real money moves — this mirrors real escrow logic with test-mode Razorpay integration.

The opportunity
Automation with n8n removes human bottlenecks from the collaboration timeline

One of the most friction-heavy moments in a collaboration is the approval deadline. If a business fails to respond after a delivered submission, the influencer is stuck. Influence Hub solves this with a self-hosted n8n automation workflow.

When a collaboration is delivered, a scheduled n8n workflow checks for overdue approvals every hour. If the business approval window has passed, n8n hits a webhook endpoint that automatically releases the escrowed payment to the influencer — without any human intervention.

A second n8n workflow verifies Instagram post submission by checking the provided URL for reachability — a practical workaround to Instagram Graph API access restrictions that typically require app review approval.

In practice: The n8n webhook endpoints are protected by a shared secret in the environment. When triggered, they update the Collaboration document state to "completed" and fire a notification to both parties — making the entire resolution cycle transparent and auditable without manual admin action.

Key Takeaways
Four engineering and product lessons from building Influence Hub end-to-end as a solo developer over a full semester.
Full-stack ownership teaches system thinking
Designing, building, testing, and deploying every layer — from Mongoose schema to React routing — forces you to understand how data flows across a real application. There is no abstraction to hide behind.
Pragmatic tradeoffs > theoretical perfection
Choosing polling over WebSockets, self-reported analytics over Graph API, and documented terminals over Docker — each decision prioritised shipping a working product over a theoretically ideal one.
Security must be baked in, not bolted on
Adding Helmet, rate limiting, JWT role verification, and .env secrets from the start prevents messy retrofitting. Security decisions made at architecture time are far cheaper than those made at deployment time.
Automation amplifies a small team's reach
One n8n workflow replaced an entire async background job system. For solo developers and small startups, automation tools like n8n close the gap between limited engineering headcount and production-grade reliability.

What comes next for Influence Hub

The platform is a working product — not just a prototype. Every stated feature has been implemented, tested, and documented. The architecture is designed to extend rather than rewrite.

The natural next steps: integrating the Instagram Graph API for verified follower counts, replacing simulated payments with a live Razorpay KYC gateway, upgrading messaging to Socket.io real-time delivery, and adding an AI-based influencer recommendation engine using collaboration history as training signal.

A rating and review system post-collaboration, multi-platform support beyond Instagram, and a React Native mobile application are the longer-horizon milestones that would take this from academic capstone to market-ready product.
